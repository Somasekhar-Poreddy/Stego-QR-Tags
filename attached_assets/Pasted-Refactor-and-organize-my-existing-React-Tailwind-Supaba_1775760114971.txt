Refactor and organize my existing React + Tailwind + Supabase project into a clean modular architecture.

-----------------------------------
GOAL:
-----------------------------------
Create a scalable "modular monolith" structure where:

- Consumer app
- Admin panel
- Shared logic

are clearly separated but live in the same project.

-----------------------------------
FOLDER STRUCTURE:
-----------------------------------

Organize code into:

/src
  /app              → Consumer app (user-facing)
  /admin            → Super Admin panel
  /components       → Shared reusable components
  /lib              → Supabase client, utilities
  /services         → API calls / business logic
  /hooks            → Custom React hooks
  /types            → Type definitions (if needed)
  /utils            → Helper functions

-----------------------------------
ROUTING STRUCTURE:
-----------------------------------

Implement routing using React Router:

Public routes:
- /                → Landing page
- /qr/:id          → Public QR contact page

Consumer routes:
- /app             → Home dashboard
- /app/my-qrs
- /app/create-qr
- /app/profile

Admin routes:
- /admin           → Admin dashboard
- /admin/users
- /admin/qrs
- /admin/orders
- /admin/products
- /admin/analytics

-----------------------------------
ROUTE PROTECTION:
-----------------------------------

Implement route guards:

1. Require login for /app
2. Require admin role for /admin

Logic:

- Get user from Supabase auth
- Check role from database (admin_users table or user role field)
- Redirect if unauthorized
